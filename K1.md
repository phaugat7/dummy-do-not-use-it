The Design — Simple and Clean
java@Configuration
@RequiredArgsConstructor
@Slf4j
public class KafkaConsumerConfig {

    private final SiteEnvConfigRepository siteEnvConfigRepo;

    @Value("${site.name}")
    private String siteName;

    @Bean
    public boolean orderConsumerAutoStartup() {
        boolean active = siteEnvConfigRepo
            .findBySiteName(siteName)
            .map(SiteEnvConfig::isActive)
            .orElse(false);

        log.info("Startup — site: {}, active: {}. Consumer will {}.",
                 siteName, active, active ? "START" : "REMAIN STOPPED");
        return active;
    }
}
java@Component
@RequiredArgsConstructor
@Slf4j
public class OrderConsumer {

    private final boolean orderConsumerAutoStartup;

    @KafkaListener(
        id = "orderConsumer",
        topics = "${kafka.topic.orders}",
        groupId = "${kafka.consumer.group-id}",
        autoStartup = false
    )
    public void consume(ConsumerRecord<String, String> record) {
        // Process order
    }

    @EventListener(ApplicationReadyEvent.class)
    public void onApplicationReady(ApplicationReadyEvent event) {
        if (orderConsumerAutoStartup) {
            KafkaListenerEndpointRegistry registry =
                event.getApplicationContext().getBean(KafkaListenerEndpointRegistry.class);
            MessageListenerContainer container =
                registry.getListenerContainer("orderConsumer");
            if (container != null && !container.isRunning()) {
                container.start();
                log.info("Order consumer started on startup.");
            }
        }
    }
}
java@Component
@RequiredArgsConstructor
@Slf4j
public class ConsumerLifecycleScheduler {

    private final KafkaListenerEndpointRegistry registry;
    private final SiteEnvConfigRepository siteEnvConfigRepo;

    @Value("${site.name}")
    private String siteName;

    @Scheduled(fixedDelay = 300_000) // 5 minutes
    public void manageConsumerLifecycle() {
        boolean active = siteEnvConfigRepo
            .findBySiteName(siteName)
            .map(SiteEnvConfig::isActive)
            .orElse(false);

        MessageListenerContainer container =
            registry.getListenerContainer("orderConsumer");

        if (container == null) return;

        if (active && !container.isRunning()) {
            container.start();
            log.info("Scheduler — STARTED consumer for site: {}", siteName);
        } else if (!active && container.isRunning()) {
            container.stop();
            log.info("Scheduler — STOPPED consumer for site: {}", siteName);
        }
    }
}
That's the entire design. Simple, no control topics, just DB + scheduler.
Will You Miss Anything?
Given your constraint that you always disable edge first, then enable GCP, the main concern is the 5-minute window. Let's walk through the switchover sequence:
Step 1: You set active = false in edge DB.
Step 2: You wait for edge consumer to stop.
Step 3: You enable GCP consumer.
The question is — how do you know Step 2 is complete? The scheduler might not have run yet. So there are two things worth considering:
1. The 5-Minute Gap During Switchover
After you disable the site in DB, orders keep piling up in Kafka (which is fine, they're not lost). But the edge consumer might still be running and processing orders for up to 5 minutes. This isn't data loss — it's the opposite. The edge might process a few extra orders after you intended to switch. If that's acceptable to you, no problem at all.
If you want a tighter guarantee, you have two practical options:
Option A — Just check the logs or add a status endpoint:
java@RestController
@RequiredArgsConstructor
public class ConsumerStatusController {

    private final KafkaListenerEndpointRegistry registry;

    @GetMapping("/consumer/status")
    public Map<String, Object> status() {
        MessageListenerContainer container =
            registry.getListenerContainer("orderConsumer");
        return Map.of(
            "running", container != null && container.isRunning()
        );
    }
}
Before enabling GCP, you just hit GET /consumer/status on the edge and confirm it's stopped. Simple, manual, but reliable.
Option B — Reduce the scheduler to 30 seconds but only during switchover:
Honestly, even 5 minutes is probably fine. Orders aren't lost either way — the worst case is edge processes a few more orders before stopping. Since you're controlling the sequence manually, you can just wait 5 minutes before enabling GCP.
2. App Restart While Disabled — Covered
If the edge app restarts and the site is disabled, orderConsumerAutoStartup returns false, consumer never starts. GCP keeps consuming. This is handled correctly.
3. DB Goes Down Temporarily
If the scheduler can't reach the DB, findBySiteName will throw an exception. The consumer stays in whatever state it was in, which is the safe behavior. Just make sure you have a try-catch:
java@Scheduled(fixedDelay = 300_000)
public void manageConsumerLifecycle() {
    try {
        boolean active = siteEnvConfigRepo
            .findBySiteName(siteName)
            .map(SiteEnvConfig::isActive)
            .orElse(false);

        MessageListenerContainer container =
            registry.getListenerContainer("orderConsumer");
        if (container == null) return;

        if (active && !container.isRunning()) {
            container.start();
            log.info("Scheduler — STARTED consumer for site: {}", siteName);
        } else if (!active && container.isRunning()) {
            container.stop();
            log.info("Scheduler — STOPPED consumer for site: {}", siteName);
        }
    } catch (Exception e) {
        log.error("Scheduler — DB check failed, consumer state unchanged.", e);
    }
}
Bottom Line
Your approach is sound. Given the operational guarantee that you always disable edge before enabling GCP, you won't have two consumers fighting over the same partitions. The only thing to be mindful of is the 5-minute window — just wait for one scheduler cycle (or check the status endpoint) before enabling GCP, and you're safe. No messages will be lost either way.
